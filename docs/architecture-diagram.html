State

Single Source of Truth

Monday.com Board

The shared project board where all task state lives. Columns: Status (To Do / In Progress / In Review / Done / Blocked), Priority (Low / Medium / High / Critical), Assignee, Type (Feature / Bug / Chore / Spike), and Context ID for A2A tracing. Humans can directly view, edit, and override anything on the board at any time.

MONDAY.COM

How It Works

1

Human Request

A team member @mentions the bot in Slack: "Build a user authentication system"

→
2

PO Breaks Down

Product Owner agent creates tasks on Monday.com: JWT setup, login endpoint, tests, etc.

→
3

Dev Picks Up

Developer agent is notified via A2A, reads the task, marks it In Progress, and works on it.

→
4

Review Cycle

Developer moves task to In Review. Reviewer agent checks work, approves or sends back feedback.

→
5

Done + Report

Task moves to Done. Scrum Master monitors progress, reports status, and nudges stuck items.
